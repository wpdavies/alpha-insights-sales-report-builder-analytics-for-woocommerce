Alpha Insights Developer Documentation

Welcome to the Alpha Insights developer documentation. This section provides technical reference for developers who want to extend, customize, or integrate with Alpha Insights.

What You Can Do

  • Extend profit calculations with custom cost logic
  • Add custom cost types for products and orders
  • Modify calculation formulas to match your business model
  • Integrate with external systems(ERP, accounting, analytics)
  • Create custom data exports programmatically
  • Build custom reports using the Alpha Insights data API
  • Automate workflows with webhooks and actions

Documentation Structure

Filters

Modify data and calculations:

  • wpd_ai_calculate_cost_profit_by_order: Complete profit calculation override
  • wpd_ai_cost_price_per_unit: Filter product cost retrieval
  • wpd_ai_custom_product_cost_options: Register custom product cost types
  • wpd_ai_custom_product_cost_default_value: Set default custom product cost values
  • wpd_ai_custom_order_cost_default_value: Set default custom order cost values
  • wpd_ai_order_shipping_cost_default_value: Filter shipping cost calculation
  • wpd_ai_order_payment_gateway_cost_default_value: Filter payment fee calculation

Actions

Hook into Alpha Insights events:

  • Order profit calculations
  • Report generation
  • Data synchronization
  • Integration updates

Functions

Helper functions for common tasks:

  • wpd_calculate_cost_profit_by_order(): Calculate order profit
  • wpd_get_cost_price_by_product_id(): Get product cost
  • wpd_get_custom_product_cost_options(): Get custom product costs
  • wpd_calculate_margin(): Calculate profit margin
  • wpd_store_price(): Format price for display
  • wpd_paid_order_statuses(): Get valid order statuses

Classes

Main PHP classes for advanced customization:

  • WPD_Order_Calculator: Order profit calculation engine
  • WPD_Data_Warehouse_React: Data fetching for React reports
  • WPD_Report_API: REST API endpoints for reports
  • WPD_Report_Filters: Filter data management

Getting Started

Where to Add Custom Code

Option 1: Child Theme functions.php

  • Best for theme-specific customizations
  • Located:wp-content/themes/your-child-theme/functions.php

Option 2: Custom Plugin (Recommended)

  • Best for site-specific functionality
  • Survives theme changes
  • More professional and portable

Option 3: Code Snippets Plugin

  • Use plugins like Code Snippets or WPCode
  • Easier management of multiple snippets

Basic Plugin Template 

<?php /** * Plugin Name: Alpha Insights Customizations * Description: Custom hooks and filters for Alpha Insights * Version: 1.0.0 * Author: Your Name */ defined( 'ABSPATH' ) || exit; // Your custom code here add_filter( 'wpd_ai_cost_price_per_unit', 'my_custom_cost_logic', 10, 2 ); function my_custom_cost_logic( $cost, $product_id ) { // Modify cost return $cost; } ?>

Common Customization Scenarios

Scenario 1: Integrate with External Cost Database

Fetch costs from external system instead of WooCommerce meta: 

add_filter( 'wpd_ai_cost_price_per_unit', 'external_cost_source', 10, 2 ); function external_cost_source( $cost, $product_id ) { $product = wc_get_product( $product_id ); $sku = $product ? $product->get_sku() : ''; if ( $sku ) { $external_cost = my_api_get_cost( $sku ); if ( $external_cost ) { return (float) $external_cost; } } return $cost; // Fallback to Alpha Insights cost }

Scenario 2: Add Business-Specific Costs

Add costs unique to your business model: 

add_filter( 'wpd_ai_calculate_cost_profit_by_order', 'add_marketplace_fees', 10, 2 ); function add_marketplace_fees( $results, $order ) { // If selling on Amazon/eBay, add their fees $marketplace_source = get_post_meta( $order->get_id(), '_marketplace_source', true ); if ( $marketplace_source === 'amazon' ) { // Amazon takes 15% + $0.99 $amazon_fee = ( $results['order_total'] * 0.15 ) + 0.99; if ( !isset($results['custom_order_costs']) ) { $results['custom_order_costs'] = array(); } $results['custom_order_costs']['marketplace_fee'] = $amazon_fee; $results['total_costs'] += $amazon_fee; $results['gross_profit'] -= $amazon_fee; // Recalculate margin $results['profit_margin'] = ( $results['order_total_ex_tax'] >0 ) ? ( $results['gross_profit'] / $results['order_total_ex_tax'] ) * 100 : 0; } return $results; }

Scenario 3: Custom Reporting Data

Export profit data to custom dashboard: 

function get_weekly_profit_summary() { $start_date = strtotime( 'monday this week' ); $end_date = strtotime( 'sunday this week' ); $orders = wc_get_orders( array( 'limit' =>-1, 'date_created' =>$start_date . '...' . $end_date, 'status' =>array( 'processing', 'completed' ) )); $summary = array( 'total_revenue' =>0, 'total_costs' =>0, 'total_profit' =>0, 'order_count' =>count( $orders ) ); foreach ( $orders as $order ) { $profit_data = wpd_calculate_cost_profit_by_order( $order ); if ( $profit_data ) { $summary['total_revenue'] += $profit_data['order_total_ex_tax']; $summary['total_costs'] += $profit_data['total_costs']; $summary['total_profit'] += $profit_data['gross_profit']; } } $summary['profit_margin'] = $summary['total_revenue'] >0 ? ($summary['total_profit'] / $summary['total_revenue']) * 100 : 0; return $summary; }

Code Standards

WordPress Coding Standards

  • Follow WordPress PHP Coding Standards
  • Use proper escaping and sanitization
  • Check user capabilities before displaying sensitive data
  • Use nonces for form submissions

Alpha Insights Conventions

  • Prefix custom functions with your plugin slug
  • Always return proper data types from filters
  • Check if classes/functions exist before using
  • Use defined( 'ABSPATH' ) || exit; in files

Testing

  • Test on staging site first
  • Test with variable products
  • Test with different order statuses
  • Test profit recalculation
  • Verify performance with large datasets
