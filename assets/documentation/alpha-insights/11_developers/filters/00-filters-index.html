Cost & Profit Calculation Filters

Filters that modify how costs and profits are calculated for products and orders.

wpd_ai_cost_price_per_unit

Purpose: Filter product cost price before profit calculations

Use Case: Dynamic cost adjustments, ERP integration, volume-based costs

Parameters: $cost_price_per_unit (float), $product_id (int)

wpd_ai_calculate_cost_profit_by_order

Purpose: Modify complete order profit calculation results

Use Case: Add custom costs, integrate external data, recalculate profit

Parameters: $results (array), $order (WC_Order)

wpd_ai_custom_product_cost_options

Purpose: Register custom product-level cost types

Use Case: Add packaging costs, labor fees, custom charges

Parameters: $custom_costs (array), $product_id (int)

wpd_ai_custom_product_cost_default_value

Purpose: Filter custom product cost values per line item

Use Case: Dynamic per-item costs, quantity discounts, customization fees

Parameters: $cost (float), $slug (string), $order (WC_Order), $item (WC_Order_Item_Product)

wpd_ai_custom_order_cost_default_value

Purpose: Filter custom order-level cost values

Use Case: Insurance, handling fees, international charges

Parameters: $cost (float), $slug (string), $order (WC_Order)

wpd_ai_order_shipping_cost_default_value

Purpose: Filter shipping cost calculation for orders

Use Case: Weight-based costs, carrier integration, regional pricing

Parameters: $shipping_cost (float), $order (WC_Order)

wpd_ai_order_payment_gateway_cost_default_value

Purpose: Filter payment gateway fee calculation

Use Case: Custom gateway fees, percentage calculations, tiered rates

Parameters: $payment_cost (float), $order (WC_Order)

Menu & Navigation Filters

Filters for customizing the Alpha Insights admin menu and navigation.

wpd_alpha_insights_menu_items

Purpose: Modify menu structure (add/remove/reorder items)

Use Case: Custom menu pages, reorganize navigation, role-based menus

Parameters: $menu_items (array)

Recommended: Best filter for structural menu changes

wpd_alpha_insights_menu_html

Purpose: Modify complete rendered menu HTML

Use Case: Add branding, custom widgets, completely custom menu HTML

Parameters: $menu_html (string)

Advanced: Use only if you need HTML-level control

wpd_alpha_insights_active_parent_menu_item

Purpose: Control which parent menu item is highlighted as active

Use Case: Custom page routing, context-based highlighting

Parameters: $menu_key (string|null)

wpd_alpha_insights_active_submenu_item

Purpose: Control which submenu item is highlighted as active

Use Case: Custom submenu pages, query-based routing

Parameters: $submenu_key (string|null), $parent_key (string), $children (array)

Price & Currency Filters

Filters for customizing how prices and currencies are displayed.

wpd_price

Purpose: Modify final price HTML output

Use Case: Color coding, tooltips, data attributes, custom formatting

Parameters: $html (string), $price (string), $args (array), $unformatted (float), $original (float)

Recommended: Best filter for price display customization

wpd_woocommerce_currency_symbol

Purpose: Modify currency symbols

Use Case: Custom symbols, cryptocurrencies, regional variations

Parameters: $symbol (string), $currency (string)

Report & Data Filters

Filters for controlling report behavior and data processing.

wpd_ai_report_filters_allowed_keys_for_query_parameter_values

Purpose: Add custom query parameters for report filtering

Use Case: Custom report filters, additional data filtering

Parameters: $allowed_keys (array)

Security: Controls which URL parameters are processed

wpd_ai_report_filters_is_transient_enabled

Purpose: Enable or disable report data caching

Use Case: Development, debugging, real-time data requirements

Parameters: $is_enabled (bool)

Performance: Disabling cache affects report loading speed

WooCommerce Compatibility Filters

Standard WooCommerce filters that Alpha Insights uses for compatibility.

View All WooCommerce Filters

Alpha Insights respects and uses these WooCommerce core filters:

  • raw_woocommerce_price - Filter raw price before formatting
  • formatted_woocommerce_price - Filter formatted price string
  • woocommerce_price_trim_zeros - Control decimal zero trimming
  • woocommerce_currency_symbols - Modify currency symbol array

Note: Changes to these filters affect both WooCommerce and Alpha Insights.

Filter Usage Best Practices

1. Choose the Right Filter

  • Use specific filters (e.g., wpd_ai_order_shipping_cost_default_value) rather than general ones when possible
  • Prefer Alpha Insights filters (wpd_*) over WooCommerce filters for isolated changes
  • Use menu structure filters before HTML filters

2. Performance Considerations

  • Many filters run on every page load - keep logic lightweight
  • Cache expensive operations (API calls, complex calculations)
  • Use early returns to minimize processing
  • Avoid database queries in frequently-called filters

3. Code Quality

  • Always return the expected data type
  • Handle edge cases (null values, missing data)
  • Check if data exists before using it
  • Use WordPress sanitization functions
  • Document your custom filters for future reference

4. Testing

  • Test with different user roles and capabilities
  • Verify with various currencies and formats
  • Check negative values, zero, and large numbers
  • Test mobile/responsive display
  • Use debugging to verify filter execution

Example: Adding a Custom Filter

/**
 * Add custom fulfillment cost to orders
 * 
 * @param float $cost Current cost value
 * @param string $cost_slug Cost type identifier
 * @param WC_Order $order Order object
 * @return float Modified cost
 */
function my_custom_fulfillment_cost( $cost, $cost_slug, $order ) {
    
    // Only apply to fulfillment cost type
    if ( $cost_slug !== 'fulfillment' ) {
        return $cost;
    }
    
    // Get order total
    $order_total = $order->get_total();
    
    // Tiered fulfillment costs
    if ( $order_total > 500 ) {
        return 15.00; // Large orders
    } elseif ( $order_total > 100 ) {
        return 10.00; // Medium orders
    }
    
    return 5.00; // Small orders
}
add_filter( 'wpd_ai_custom_order_cost_default_value', 'my_custom_fulfillment_cost', 10, 3 );

Getting Help

Documentation

  • Each filter has detailed documentation with multiple examples
  • Check related filters for similar functionality
  • Review "Related Functions" section for helper functions

Debugging

  • Use error_log() to log filter values
  • Set filter priority to 999 for debugging hooks
  • Check WordPress debug.log file for errors
  • Use browser developer tools for HTML/CSS issues

Common Issues

  • Filter not working: Check filter name spelling and priority
  • Wrong data type: Ensure you return the correct type (float, string, array)
  • Performance issues: Move expensive operations outside the filter
  • Conflicts: Check for other plugins using the same filters
